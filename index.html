Prework Study Guide

HTML

  • The head element contain information about the webpage.
  • The body element represents the visible content shown to the user.
  • The charset element sets our character encoding
  • The http-equiv element sets an HTTP header for content
  • The viewport element sets the default view of the webpage
  • The P element is paragraph
  • The img element contains info about images displayed
  • src attribute is the source and defines the location of an image
  • The alt attribute is for screen readers and describes the appearance and functioanlity of the image
  • The br element creates a line of empty space between to blocks of content
  • The a element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link
  • Using Git commands, we can initiate (or create) a repo, add our code to the repo, and view the different versions of our code, which are each stored in a unique commit.
  • Git is great for tracking changes locally on your own computer, but to collaborate with others, you need to use another tool, GitHub

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • be sure to link the external css sheet to the index file utilzing the rel and href commands
  • Declarations contain two important components: the CSS property we want to apply and the value of the property

Git

  • Git is a version control system that lets you manage and keep track of each version of your project
  • Git commands are not the same as the bash commands, Git commands always start with git and are used to save and track versions of your code
